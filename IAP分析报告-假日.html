三、二维评级矩阵

将用户按 D1留存 × ARPPU 和 D1留存 × CPA 分别划分为 3×3 共9个格子，每个格子独立计算 D360 ROI 和评级。

3.1 D1 × ARPPU 矩阵

分桶：D1低(<7.6%) / 中(7.6~12.1%) / 高(>12.1%)  |  ARPPU低(<20.07) / 中(20.07~22.63) / 高(>22.63)

核心洞察：
• D1低 → 全红：无论用户多能付费（ARPPU高），D1低就是不行。留存是变现的前提。
• D1中 + ARPPU低：差一点过线（92%），建议观察。
• D1中 + ARPPU中/高 → 绿色：中等留存+中等以上付费能力即可回本。
• D1高 → 全绿：高留存用户无论付费高低都能回本（靠长尾回收）。

3.2 D1 × CPA 矩阵

分桶：D1同上  |  CPA低(<2.57) / 中(2.57~3.55) / 高(>3.55)

核心洞察：
• D1中 + CPA低 = 152% ROI：这是最具操作意义的“甜区”——用低成本获取中等留存用户，回本确定性高。
• D1高 + CPA低 = 509% ROI：顶级组合，但量可能有限。应该优先抢这类流量。
• D1中 + CPA高 = 105% ROI：刚好过线，需谨慎。
• D1低 → 全红：即使CPA极低也救不了。建议果断放弃低D1留存流量。
使用建议： 日常投放决策中，先看 D1×CPA 矩阵（因为CPA可以实时观察）， 再辅以 D1×ARPPU 矩阵做长线质量判断。两个矩阵任何一个格子为红色即谨慎投放。

五、放大系数参考

用户分档定义

分档对应桶D1留存范围特征
低质用户Q1+Q20.3% ~ 8.0%早期ROI低，但长尾贡献大，放大系数偏大
高质用户Q3+Q4+Q58.0% ~ 49.3%早期ROI已较高，后续空间小，放大系数偏小
为什么低质用户放大系数更大？ 低质用户早期大量流失，D7 ROI 分母小；留下的少数人长线付费强，D360 ROI 被拉高。高质用户 D7 ROI 已经不错，后续空间相对有限。好比起跑慢但耐力好的人，后半程/前半程比例自然更大。
